# 03 · 模组抽象契约（v3：从四个模组的数据里长出来）

> 最终目的："所有模组都能抽象，让 AI 主持人推进游戏。"
>
> **本分支单干整条链**：模组原文 → 预处理 agent → 契约 → 主持人 agent →
> 导演层。契约是这条链的窄腰——两个 agent 之间的接口。
>
> **v3 的来历（2026-07-24）**：v1/v2 的契约都是从《追书人》一个模组倒推的，
> 是"被设计"的。v3 之前跑了一轮**契约发现实验**：4 个结构不同的模组
> （宅邸调查 / 封闭生存 / 封闭倒计时 / 城市多线），用**不预设类型**的探针
> 抽取实体与关系，从数据里归纳。本文是那轮实验的收敛。
>
> ⚠️ 证据边界：4 个模组**全是 COC、全是中文（翻译）本、篇幅 1.4–3.8 万字**。
> 战役级、规则书内嵌模组、其它 TRPG 系统均未验证。下面每一条抽象都要能
> 说出"哪个模组的数据逼出了它"，说不出来的就是猜的。

---

## 一、三层抽象（本文的骨架）

契约发现实验把观察逼到了三个高度。**高层负责"不跑偏"，低层负责"能落地"**，
三层缺一不可。

```
顶层  World = ⟨Facts, Visibility, Transition⟩     ← 北极星，一句话
中层  三重投影：排版轴 / 知情权轴 / 时间选择轴     ← 契约要表达什么
底层  八类关系                                     ← 实现时的对照表
```

### 顶层：可被主持的世界，只有三个基本面

把 COC、模组、克苏鲁全部抽掉，剩下的是：

> **一个世界 = ⟨一组事实, 观察者→他那份事实子集的映射, 事实随行动演化的转移函数⟩。**
>
> **主持 = 逐轮**：接收行动 → 算转移 → 对每个观察者投影出他该看到的新事实。

- **Facts（有什么）** —— 实体、地点、线索、数据。静态，永远为真。
- **Visibility（谁知道）** —— 同一组事实，不同观察者各看到一个子集。
  静态结构，但**分观察者**。
- **Transition（会变成什么）** —— 事实随玩家行动（和时间）演化。动态。

**为什么停在这里**：再抽一层就是"世界 = 状态 + 转移"的通用状态机理论，
不再含"主持 / 秘密 / 多人"的任何信息，对我们零指导。第三层还保留着
**Visibility**——而 Visibility 恰恰是"主持一个 TRPG"区别于"跑一个状态机"的
**唯一本质**：多人各自持有不同的事实子集，主持人的核心工作就是管理这些子集
之间的信息流。所以 ⟨F, V, T⟩ 是**有用的最高点**。

### 中层：模组文档是游戏世界的三重投影

模组是一份**写给人读的线性文档**，要描述的却是一个**非线性的游戏世界**。
它必然沿三个轴把三维的世界"压平"成一维的文本——三个轴一一对应顶层三面：

| 压扁沿的轴 | 压平的结果 | 逆投影还原出 | 对应顶层 |
|---|---|---|---|
| **排版轴** | 一个实体被拆成多块写（外观/行为/数据/规则分列） | **实体表** | Facts |
| **知情权轴** | 同一事实写成公开版 + 真相版两份 | **密级配对** | Visibility |
| **时间选择轴** | 玩家的选择空间被写成章节顺序 | **状态机** | Transition |

**于是预处理 agent 的任务从"把模组塞进 schema"变成"做三次逆投影"。**
这个转变的实际价值是：每次逆投影都**可机械验证**——

| 逆投影 | 可验证的判据 | 防的失败 |
|---|---|---|
| 片段 → 实体表 | 每个片段都归属某实体，**无孤儿片段** | 信息散落 |
| → 状态机 | 每个状态可达、**每个选择都有后果** | 死路 |
| → 密级配对 | 每个 KP 版事实要么有对应玩家版，要么显式标"永不公开" | 漏密 |

这比"schema 校验通过"强：schema 校验只证明**形状**对，这三条证明
**没丢东西、没死路、没漏密**。

### 底层：八类关系（四模组数据归纳）

裸抽取产出的是扁平片段（实测 59 条片段之间 id 互引 = 0——**只做了排版切分，
一次逆投影都没做**）。关系发现探针补出片段之间的联系，去重归纳成八类，
按它们属于哪次逆投影分组：

| # | 关系类型 | 属于 | 四模组表现 | 现 schema |
|---|---|---|---|---|
| 1 | **聚合**（多片段定义一个实体；高频子类：叙事身份↔游戏数据） | 实体表 | 全部四个，大量 | ❌ |
| 2 | **层级/细化**（总述↔具体） | 实体表 | 全部四个，最高频 | ❌ |
| 3 | **包含**（地点↔其中物件/子区域） | 实体表 | 全部四个 | ⚠️ 只能塞 `sub_node` |
| 7 | **规则↔适用**（叙事内容↔该处生效的机制） | 实体表 | 全部四个 | ❌ |
| 5 | **视角配对**（同一事实的公开版↔真相版） | 密级配对 | 调查型强、生存型弱；**有模组直接把 KP/PC 信息写成配对两块** | ❌ |
| 4 | **时序/因果**（A 之后/导致 B） | 状态机 | 生存型、线性型强 | ✅ `leads_to` |
| 6 | **并列选项**（一个情境→N 个平行应对） | 状态机 | 生存型多逃生路径、城市型攻击vs说服 | ⚠️ `branches` |
| 8 | **路径×遭遇**（沿某路径行进时的条件遭遇） | 状态机 | 生存型独有 | ❌ |

**关键裂缝**：现有 `ModuleNode` **同时充当实体容器和状态节点**——想说"这房间
里有哪些材料"要塞 `sub_node`，想说"从这走到哪"要用 `leads_to`，想说"这段
描述和那块数据是同一个怪物"根本没地方放。实体表和状态机被挤进了一个字段，
所以两边都不好用。**这就是 id 互引为 0 的根因**：文本片段和游戏实体本非
一对一，而现有 schema 假设它们是。

## 二、契约发现实验的硬数据（可追溯）

探针与产物在 gitignored 的 `模组资料/`；脚本在 `trpg-backend/scripts/module_probe/`
（进 git，方法可审计）。

| 模组 | 结构类型 | 片段数 | 非空行覆盖 | 关系交集率（两遍对拍） |
|---|---|---|---|---|
| 科比特先生 | 宅邸调查 | 59 | 97.4% | 51% |
| 神秘渡轮 | 封闭+倒计时 | 43 | 91.7% | 43% |
| 复足 | 封闭生存/战斗 | 50 | 93.1% | 60%→**72%**（分批后） |
| 死者的顿足舞 | 城市多线 | 116 | 87.6% | 25%→**73%**（分批后） |

### 结论 1：关系稳定的天花板 ≈ 73%

两遍独立跑（temp 0.3 / 0.5）机械对拍，四模组分批后收敛到 72–73%：约四分之一
的关系对只出现一遍。**这量化了"关系类型不稳定"到底多严重。**

→ **契约里关系不能用自由文本。** 自由文本适合"内容"，不适合"结构"，而关系
是结构（代码要拿它做遍历、可达性、密级配对，骨架层必须稳定）。但也不能退回
拍脑袋的枚举——正解是 **有限类型集（就是上面八类）+ 自由文本说明**：类型从
数据归纳、稳定可遍历，原话作为说明、不丢信息。

### 结论 2：容量是关系发现的真瓶颈

顿足舞 116 片段一次性问关系，交集率崩到 25%；分批（每批 30 + 全量 id/title
简表）后升到 73%。对照组复足（50 片段）分批只从 60% 微升到 72%——
**条目越多受益越大**，正是"注意力容量溢出"该有的特征。若分批是无差别加分，
两者该涨得一样；差异模式本身就是根因的证据。

### 结论 3："多线并行"不是 schema 缺口

顿足舞按"多线"特征选入，但抽出的关系里**没有任何"支线归属"表达**。因为
多线是玩家在**状态机（T）上走出的轨迹**，不是**文档（F）的结构**。
→ 印证顶层抽象：属于 T 的东西不会出现在 F 的投影里。选它压测 schema 形状
这一点判断错了，它验的是运行时状态、不是文档结构。

## 三、⟨F,V,T⟩ 作为试金石：回验已有的每个决定

一个真的抽象，应能解释你不知道它时做对的事，也能指出做错的。逐条验：

| 已做的决定 | 在 ⟨F,V,T⟩ 里是什么 | 判定 |
|---|---|---|
| keeper 两阶段（裁决/叙事） | 裁决=算 T；叙事=对观察者做 V 投影 | ✅ 架构天然对齐，不是凑的 |
| `keeper_state` | T 的当前状态 | ✅ |
| 议程 / 世界心跳 | T 有**不依赖玩家行动**的自发项 | ✅ 解释了心跳为何是刚需 |
| 泄密红线 / 分区层 | 防 V 投影出错 | ✅ 整个实验最防的就是它 |
| 掷骰 | T 里的非确定性转移 | ✅ |
| "多线不在关系里"（当时的判断） | 多线是 T 上的轨迹，非 F 的结构 | ✅ 判断正确 |

七个决定，无一打脸，各自归位。

## 四、⟨F,V,T⟩ 给出的预测：最弱的一项是 V

三元组里，Facts 有实体表、Transition 有状态机+keeper_state，**Visibility 现在
只是"每块内容打一个 audience 静态标签"——粗粒度、不分观察者、不随时间变。**
而真实的 V 是：

1. **分观察者**：玩家 A 挣得的线索，B 未必知道（现在是"对玩家整体可见"，
   无 per-player 视角）；
2. **随时间翻面**：同一事实，挣得前 KP 独占、挣得后转公开（"视角配对"说的
   正是这个，但现在没有"翻面"机制）。

→ **V 不该是标签，该是一个随 T 演化的、per-observer 的函数。** 这是三元组
逼出来的、此前没意识到的最大结构性缺口——**比导演层三提案更根本**。
（不在本轮做，记在这里当路标。）

## 五、契约的实现分层：另一根正交的轴（"多强类型"）

上面三层回答"契约表达什么"（按文档怎么产生分）。**这一层回答"每部分该多强
类型"（按谁消费分）**，与上面正交，并存。

- **骨架层（代码消费，强约束）**：凡代码要拿来记账/去重/查表/防幻觉的，必须
  可寻址——唯一 id、引用闭合、技能名可查 COC7 规则表、NPC 专有名词表。
  代码对这层只**发号/记账/去重/留痕/查表，不理解语义**。
  → 八类关系属于这一层（结论 1：有限类型集，稳定可遍历）。
- **内容层（LLM 消费，一律自由文本）**：动机、氛围、KP 指引、检定后果——
  不枚举、不归一化。主持人手里有全文+历史+状态，语义判断该它做；代码去算
  反而脆（实测把"第2天傍晚"当"第2个夜晚"就误触发议程）。
- **分区层（安全约束，唯一必须严格的语义要求）**：玩家可见 / KP 绝密。分错了
  主持人再谨慎也泄密。**这一层正是 V**，也是结论指出的最弱项。

**契约厚度的定则**：由「代码机械操作的必需品」定下界，不由「主持人想要什么」
定上界。**强类型要付费**——每加一个枚举字段就问："预处理 agent 填得出来吗？
填错多贵？"填不出来的退自由文本。（提案①把 `agenda.trigger` 钉成
`game_night/silence/manual` 三枚举，就是从一个模组倒推、把语义归类难题
转嫁给预处理 agent 的反面教材，已列入路线第 3 步收缩。）

## 六、预处理 agent：形态、职责、验收

### 形态：离线流水线，不是 tool-calling loop

keeper 是回合制交互、要快；预处理是**离线批处理、可以慢**——可多轮自检、
可人工介入、可两遍对拍。

```
原文 → 段落重组（消除 PDF 视觉行伪影，无损校验）→ 分块 → 逐块抽取 →
合并 → 校验 → 自修 →（人工抽检）→ 结构化 JSON
```

### 职责：三次逆投影（= 中层）。**不做的是"塞进预设 schema"。**

### 复用已验证的手法

- **两遍独立抽取 + 对拍**（#114 扩 229 职业用过：64 条分歧里 60 条代码机械
  消解，真人裁定仅 4 条）。关系发现天花板 73%，对拍是"关系可信度"的验收手段。
- **大模组关系发现必须分批**（结论 2）：每批约 30 片段 + 全量 id/title 简表，
  跨批关系不丢。

### 可机械验收清单

| 验收项 | 手法 | 防的失败 |
|---|---|---|
| 引用闭合 | 悬空引用扫描（#114 槽候选守卫先例）；**pass1 真的引用过清单外 id，被挡下——不是理论顾虑** | 运行时静默失效 |
| id 唯一 | 平凡 | 记账串味 |
| 技能名合法 | 对 COC7 规则表 | 检定发不出来 |
| **内容覆盖率** | **只看非空行覆盖 + 漏掉字符占比 + 最长连续缺口**（总行覆盖率被空行污染，是伪指标——实测空行占比 16%→29% 时总行覆盖假降 10 个点，非空行其实 88–97%、漏的全是散点） | 静默丢整章 |
| 秘密分区 | 真相关键词在玩家可见区零出现 | 泄密 |

### 一条必须接受的现实

真实模组写作质量差异极大，有的原文根本没有清晰检定点或结局条件。预处理不能
无中生有。→ **契约必填项要极少**，主持人要能在信息不全时工作（它已经能——
`kp_guidance` 就是自由文本兜底）。

## 七、路线（v3）

```
✅ 0. 四个结构不同的模组，裸抽取 + 关系发现探针跑完，数据归纳成本文
→  3. 收缩现有 keeper schema 到契约：agenda.trigger 退自由文本、
      silence 挪出模组 schema（属引擎侧心跳，非模组时间轴）
   4. 预处理 agent 加契约约束 + 校验闭环（生成→校验→自修），先在《科比特
      先生》上跑通"逆投影"完整链
   5. per-observer / 随时间翻面的 Visibility（第四节预测，最根本的缺口）
   6. 回到导演层（提案②世界心跳 / ③阶段状态机）
```

**为什么第 3 步现在就能做**：`trigger` 收缩方向已定、独立于其它，且同时修掉
"代码去算第几晚"那个脆点。它是本轮契约定稿**直接解锁**的最小可执行步骤。

**为什么导演层排最后**：心跳吃议程当燃料，且它属于 T 的自发项——T 的表达
（状态机）没定型就做心跳是白做。

## 八、推进原则

- **数据驱动，代码不认模组**：任何模组专属逻辑想写进 `keeper/*.py`，就是抽象
  失败的信号。
- **契约是被发现的，不是被设计的**：新形状从模组数据里长出来，不从"最近看过
  的那个模组"倒推。第二个模组**绝不能人工结构化**（会迁就已有 schema，得到
  自我实现的预言）——让预处理 agent 裸跑，看它自然吐出什么。
- **强类型要付费**：见第五节定则。
- **契约的每一条都要能说出"哪个真实模组逼出了它"**。说不出来的，就是猜的。
- **抽象分层各司其职**：⟨F,V,T⟩ 管"不跑偏"、三重投影管"表达什么"、八类关系
  管"怎么落地"。别拿顶层去写代码（它不给实现），也别拿底层谈方向（见木不
  见林）。
